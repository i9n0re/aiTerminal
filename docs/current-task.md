# Current Task

## Goal
- Fix mobile frontend behavior where the virtual keyboard covers the terminal input line.
- Prevent Backspace from navigating the Mac app/WebView away from the terminal when no input has focus.
- Prevent tmux touch scrolling from being mistaken for a keyboard-open tap.
- Make the terminal background pure black.
- Use Monaco as the first terminal font.
- Keep the right drawer window list bound to the startup tmux target.

## Current State
- Implemented mobile keyboard avoidance in `html/src/components/terminal/index.tsx`.
- The terminal root now uses the current `visualViewport` width, height, and offsets, plus an estimated keyboard inset when needed.
- Samsung Chrome showed a gap where terminal rows only moved after the first typed character. A later pass briefly reserved estimated keyboard height immediately on touch, but that blocked the first keyboard popup.
- Latest pass now focuses xterm on terminal touch, delays the estimated keyboard inset until after browser focus handling, and subtracts any real viewport shrink to avoid double shrinking.
- Latest pass tracks whether a real keyboard viewport signal happened; when the viewport returns to full height, it clears the inset so the page drops back after keyboard dismissal.
- Added terminal focus/touch handling, VirtualKeyboard API observation when available, repeated xterm refits during keyboard animation, and a mobile fallback inset when the browser does not report keyboard shrink.
- Added `interactive-widget=resizes-content` to `html/src/template.html`.
- Color palette was restored to the original xterm theme after the Codex-like palette was too bright.
- Sidebar and quick-key controls now position relative to the resized terminal root.
- Added a global Backspace guard in `html/src/components/terminal/index.tsx`: when focus is not inside an editable element, Backspace is prevented from triggering browser/WebView history navigation and focus is returned to xterm.
- Touch scrolling now marks the gesture as moved, cancels pending keyboard fallback, and skips the tap-triggered keyboard inset on `touchend`.
- Changed xterm theme background from `#2b2b2b` to `#000000` in `html/src/components/app.tsx`.
- Changed xterm `fontFamily` to `Monaco,Menlo,Consolas,Liberation Mono,Courier,monospace`.
- Backend tmux list/select commands now parse the startup command's `tmux ... -t <target>` and run against that explicit target, so the drawer does not drift to the most recently used tmux context.
- Frontend window-list parsing now accepts colons in tmux window names and clears the list when a list response contains no windows.
- `src/html.h` was regenerated by the frontend build.
- Backend binary `build/ttyd` has been rebuilt successfully.
- Port `8888` is listening from rebuilt `build/ttyd` PID `49335`, hosted in tmux session `ttyd-8888`; the served command is `env -u TMUX tmux a -t gemini`.
